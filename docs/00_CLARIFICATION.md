# 00 - Clarification v13：清除舊資料，確保媒體完整性

## 現況

v5.0 部署後，69/76 個 OSS 媒體 URL 回傳 404（檔案不存在）。原因是 v9 OSS 遷移時清理了本地媒體，但 Telegram 下載器未完整重跑所有頻道，導致 messages.jsonl 中有大量指向不存在檔案的 URL。

## 可行方案

### 方案 A：全清 + 強制重新下載 ★推薦

刪除所有 messages.jsonl 和 .downloaded_state.json，觸發下載器從頭爬取所有頻道。

- ✅ 最乾淨，保證所有 URL 都有效
- ✅ 一次性解決所有不一致問題  
- ❌ 需要等待下載器跑完所有頻道（取決於 Telegram rate limit）
- ❌ 過程中網站暫時沒有內容

### 方案 B：驗證過濾

generate_html.py 生成前檢查每個 media URL 是否存在（HEAD 請求），過濾掉失效的媒體。

- ✅ 不影響現有下載流程
- ❌ 每個 URL 發一次 HTTP HEAD，生成時間大幅增加
- ❌ 只是掩蓋問題，不解決根源

### 方案 C：增量修復

只刪除 media path 為 404 的 message，保留文字和有效媒體。

- ✅ 精準修復，不丟失文字內容
- ❌ 需要逐條驗證 URL

## 確認狀態

| 項目 | 狀態 |
|------|------|
| 方案 | 推薦 A：全清重下 |
| 確認開始 | ⏳ 等你確認 |
