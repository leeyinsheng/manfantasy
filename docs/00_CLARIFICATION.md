# 00 - Clarification v11：OSS 防盜鏈

## 我對目標的理解

目前 OSS bucket 是 public-read，任何人拿到媒體 URL 就能直接存取（例如分享給別人、嵌入其他網站）。你希望媒體檔案只能透過你的網站播放，不允許直接訪問或外部引用。

最終結果：用戶只能在 http://8.213.209.231/dream/ 上看到圖片/影片，直接打開 OSS URL 或在其他網站引用會被拒絕。

## 可行方案

### 方案 A：OSS 防盜鏈（Referer 白名單）★推薦

阿里雲 OSS 內建 Referer 防盜鏈功能。在 bucket 設定中，只允許來自你網站的請求通過。

- 配置：OSS 控制台 → 防盜鏈 → 白名單加入 `http://8.213.209.231`
- ✅ 零程式碼改動，靜態 HTML 完全相容
- ✅ OSS 層級攔截，不消耗伺服器資源
- ✅ `<img>` `<video>` 請求自動帶 Referer，瀏覽器行為天然支援
- ❌ 可直接繞過（curl 偽造 Referer），但不是主要威脅

### 方案 B：OSS 簽名 URL + 短期有效

每次生成 HTML 時，為每個媒體 URL 附加過期簽名。簽名 URL 在指定時間後失效。

- ❌ 需要動態生成簽名（目前 generate_html.py 是靜態的）
- ❌ 每次 HTML 重建時需重新簽名所有 URL
- ❌ 用戶長時間停留頁面後，影片可能過期無法播放

### 方案 C：Nginx 反向代理

Nginx 接收媒體請求，檢查 Referer，轉發到 OSS。

- ❌ 所有媒體流量經過伺服器（失去 OSS 分流優勢）
- ❌ 增加伺服器頻寬成本

## 我的假設

1. **方案 A 優先**。OSS 防盜鏈最簡單，符合靜態 HTML 架構。

2. Referer 白名單設定：`http://8.213.209.231`（Allow Empty Referer 也建議開啟，確保 `<video>` 標籤和直接輸入網址都能正常）

3. 不需改任何程式碼。
