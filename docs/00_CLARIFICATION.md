# 00 - Clarification v12：小紅書風格全面改版

## 我對目標的理解

目前 v7 是深色主題的 waterfall 卡片，與小紅書的明亮簡潔風格差距大。你希望 UI/UX 完全比照小紅書 App。

## 小紅書 vs 現狀

| 設計元素 | 現狀 (v7) | 小紅書 |
|---------|-----------|--------|
| 底色 | 深色 #0a0a0a | 白色/淺灰 |
| 搜尋 | 🔍 圖示點開才展開 | 頂部常駐搜尋欄 |
| 卡片 | 無邊框、無頭像 | 白色卡片、作者頭像、愛心數 |
| 導覽 | 底部圖示 only | 底部圖示 + 文字 |
| 分類 | 無 | 頂部水平滾動分類標籤（推薦/穿搭/美食...） |
| 卡片間距 | 0.6rem | 更緊湊 |
| 字體 | 系統預設 | 特定字級/行高 |
| 詳情 | 底部 sheet | 全螢幕筆記詳情頁 |

## 我的假設

1. **僅改 UI 層**（CSS + HTML 結構 + JS），不改資料層

2. **小紅書核心特徵**：
   - 白色底 + 淺灰卡片
   - 每張卡片：大圖 + 標題（最多 3 行）+ 底部頭像 + 愛心數
   - 頂部搜尋欄 + 分類標籤橫向滾動
   - 底部導覽：圖示 + 文字（5 tab）
   - 點擊卡片 → 全螢幕筆記頁（圖片輪播 + 文字 + 留言區佔位）
   - 瀑布流仍有，但卡片比例更接近 3:4

3. **範圍界線**：僅改 generate_html.py 的 CSS/HTML/JS。不涉及後端。

4. **手機直向優先**：與 v7 一致，桌面版置中顯示手機寬度。

## 盲點

1. **淺色主題確定？** 小紅書是白色底，跟目前深色完全不同。

2. **分類標籤**（推薦/穿搭/美食...）：要對應到現有 Telegram 頻道群組，還是自定義？

3. **愛心數/留言數**：Telegram 沒有這些資料，是否用假資料佔位？

4. **詳情頁互動**：小紅書是整頁捲動（圖片輪播在上、文字在下），要實現到什麼程度？
